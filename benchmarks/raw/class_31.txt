<p><br/><font color="maroon"><a name="overview"><b>Overview</b></a></font>
</p><ul>
<p>Welcome to Software Construction! 
In this course, we will study a number of interesting topics including
software processes, requirement engineering, software architecture and 
component design, software testing, debugging, and
software maintenance.
<br/><br/>
This course assumes familiarity with basic programming skills
and data structures. 
Some background in object-oriented programming (e.g., C++ or Java) is
not required, but would be good to have.
You will need to program in a 8 people team. 
</p>
</ul>

<p><br/><font color="maroon"><a name="project"><b>Projects</b></a></font>
</p><ul>
<p>This is a <b>tentative</b> version.</p>
<p>
You will do a warm-up project in a group of two with the due day on <b>10/14</b>. 
More information about this
warm-up project will be posted before the second lecture of the class.
</p>
<p>
You will propose a project and develop it following the process learned 
in the class with your teammates.<br/>
Your project will go through the following milestones, with every milestone being graded.<br/>
More details will be announced later.<br/>

</p><ul>
<li>Milestone 1 (10/17): proposing a project, with requirement details and user stories written down (group of 2--3 people);
</li><li>Milestone 2 (10/31): requirement validation and project planning, with development team and plans set up (group of 8 people);
</li><li>Milestone 3 (11/06, 11/13): 1st development iteration, with unit test cases to show progress (group of 8 people);
</li><li>Milestone 4 (11/20, 11/27): 2nd development iteration, with unit test cases to show progress (group of 8 people);
</li><li>Milestone 5 (12/04): system testing and documentation (group of 8 people);
</li><li>Milestone 6 (12/09): acceptance testing (groups testing each other's projects).
</li></ul>
</ul>
