The release also includes concurrency API for Intel's Pin dynamic instrumentation tool, a Pin based implementation of <a href="http://research.microsoft.com/en-us/downloads/b23f8dc3-bb73-498f-bd85-1de121672e69/"> CHESS</a>, and a happens-before data-race detector.

<br/>
<br/>
<li class="MsoNormal" style="margin-right: 2.25in;"><span style="">
   A collection of <a href="http://web.eecs.umich.edu/~jieyu/bugs.html">concurrency bugs</a> that we used in <a href="http://www.eecs.umich.edu/~nsatish/papers/OOPSLA-12-Maple.pdf"> OOPSLA 2012 paper</a>.

<br/>
<br/>
<li class="MsoNormal" style="margin-right: 2.25in;"><span style="">
  Click <a href="http://www.cs.ucla.edu/~todd/research/llvmsc.tar.gz"> here </a> to download SC-preserving version of the LLVM compiler. For details refer to our <a href="http://www.eecs.umich.edu/~nsatish/papers/PLDI-11-SC-Preservation.pdf"> PLDI 2011 paper</a>. 

<br/>
<br/>
<li class="MsoNormal" style="margin-right: 2.25in;"><span style="">
<a href="http://software.intel.com/en-us/articles/program-recordreplay-toolkit">PinSEL</a> 
  -- a record-and-replay tool.
  
  It implements the 
  <a href="http://www.cs.ucsd.edu/~calder/abstracts/ISCA-05-BugNet.html">BugNet</a> 
  checkpointing/logging scheme using the 
  <a href="http://rogue.colorado.edu/pin/index.html">Pin</a>
  dynamic instrumentation framework. PinSEL can record and replay any
  multi-threaded program across different system environments.   Currently, it
  is used as a front-end for application level architectural simulators to avoid 
  emulating the system effects. 
  
  For more details refer to our <a href="http://www.cs.ucsd.edu/~calder/abstracts/SIGMETRICS-06-PinSEL.html">
  SIGMETRICS 2006 paper.</a>
